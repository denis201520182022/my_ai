Проект ИИ агент

Цель проекта:
Создать полезного ИИ агента, опробовав на нем определенный стек технологий и подходов

Что должно в нем быть обязательно:
Универсальный интерфейс для подключения любых llm (openai/gemini/openrouter)
Написан на Langchain и Langraph
Подключен Langphuse (LangSmith)
Наличие RAG
Использует tools
Доступ в итернет через дакдакго
Принцип Re Act - цепочка запросов и шагов, а не один запрос к llm и ответ


Полный мониторинг вокруг ИИ агента: трассировка, отслеживание на каждом шагу, какие данные приходят в модель, какие выходят, латенси, токены и стоимость, колочество ретраев - антицикл бесконечности (вроде все это через langsmith)



Функциональные требования:

Чат в телеграмме
Должен уметь принимать голосовые сообщения
Должен уметь искать в интернете
Должен иметь память (возможность запомнить надолго какую то информацию)
Должен уметь искать по RAG
Должен уметь создавать напоминания (я прошу напомнить о чем то в определенное время - он сам когда нужно напоминает)


Легенда:
Это многофункциональный личный помошник
в rag я могу ему данные на любую тему подсовывать
То есть он как секретарь, может поискать найти, помнит важное, напоминает мне когда нужно о чем то



Архитектурные требования:
Система должна быть распределенной ассинхронной
Система должна быть масштабируемой: чтобы мог добавить различные инструменты, сценарии
Взаимодействие с llm и телеграммом через прокси. В енв переменные:
SQUID_PROXY_HOST=38.180.203.212
SQUID_PROXY_PORT=8787
SQUID_PROXY_USER=zabota
SQUID_PROXY_PASSWORD=zabota2000

Все в докере (вся инфраструктура тоже)
 Миграции через алембик


Логика работы:
Пользователь пишет сообщение
Входной узел классифицирует запрос (в json отвечает и на основе него дальше выполняются действия), далее вызываются tools, получаем результат, генерируем ответ пользователю



Несколько уточнений
Лангсмит платный поэтому будем использовать опенсоурс аналог - лангфьюз
Все что пишет и бот и человек сохраняется в историю в БД
Для каждого пользователя в БД свой диалог и свои данные
Обязательно вводим защиту, бот должен быть доступен только определенным чатам (перечень id)

Пользователей в енв пропишем.
Запоминать должен либо насильно при команде слешевой, либо понимать фразы "запомни ..."

Тг интерфейс и логика движка должны быть разделены по разным независимым воркерам
Связи через очереди RabbitMQ
Для эмбедингов используем cointegrated/rubert-tiny2

    ├── Dockerfile                # Единый докерфайл для 3-х воркеров (запускаются разными командами)

    │
