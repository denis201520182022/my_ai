🚀 Руководство "Start & Go" по ИИ-Агенту
1. Подготовка окружения
Перед запуском убедитесь, что на сервере установлены:
Docker и Docker Compose
Git

Шаг 1: Настройка конфигурации
Создайте файл .env в корне проекта (используйте предоставленный ранее шаблон).
Обязательные поля:
BOT_TOKEN: Получите у @BotFather.
ALLOWED_USERS: ID вашего профиля (узнать можно через @userinfobot).
LLM_API_KEY: Ключ OpenAI или OpenRouter.
LANGFUSE_...: Ключи из cloud.langfuse.com.
PROXY_URL: В формате http://user:pass@host:port.


Шаг 2: Проверка файлов
Убедитесь, что структура папок соответствует ТЗ и файл app/ai_worker/prompts/system_prompt.md создан и наполнен.
2. Деплой (Запуск)
Запуск всей инфраструктуры выполняется одной командой:
code
Bash
docker-compose up --build -d
Что произойдет при первом запуске:
Базы данных: Поднимутся Postgres, RabbitMQ и Redis.
Миграции: Контейнер migrations создаст таблицы в БД и активирует расширение vector.
Загрузка моделей:
stt-service скачает веса Whisper (~150MB).
ai-worker скачает модель эмбеддингов rubert-tiny2 (~100MB).
Воркеры: Бот и ИИ-движок начнут слушать очереди.
3. Мониторинг и Логи
Просмотр логов
Для отладки конкретного сервиса используйте:
code
Bash
# Логи ИИ-движка (самое интересное тут)
docker logs -f agent_ai_worker

# Логи Телеграм-бота
docker logs -f agent_tg_bot

# Логи расшифровки голоса
docker logs -f agent_stt
Визуальный мониторинг (Langfuse)
Зайдите в свой аккаунт на Langfuse.
В разделе Traces вы увидите каждый запрос.
Кликните на трейс, чтобы увидеть:
Цепочку ReAct (сколько раз вызывались инструменты).
Затраты (Cost) в $ за каждый запрос.
Latency (время раздумий агента).
Точное содержимое Prompt, отправленного в модель.
Управление очередями (RabbitMQ)
Зайдите по адресу http://ваш_ip:15672 (логин/пароль из .env). Здесь можно увидеть, сколько сообщений стоит в очереди, если агент не справляется.
4. Как пользоваться
Начало: Напишите /start боту.
Голос: Просто отправьте ГС. Бот расшифрует его и передаст агенту.
Память:
Скажите: "Запомни, что мой пароль от Wi-Fi 12345" или "Мою кошку зовут Муся".
Спросите: "Как зовут мою кошку?" или "Найди в моих записях пароль".
Напоминания:
"Напомни мне выпить витамины через 2 часа".
"Напомни завтра в 9 утра позвонить маме".
Интернет:
"Какой сейчас курс биткоина?" или "Найди последние новости про нейросети".
5. Поддержка и миграции (Alembic)
Если вы решите изменить структуру таблиц в models.py:
Создайте новую миграцию:
code
Bash
docker exec -it agent_tg_bot alembic revision --autogenerate -m "added_new_field"
При следующем docker-compose up контейнер migrations автоматически применит её.
6. Масштабирование и Улучшения
Как масштабировать (если бот стал тормозить):
Если пользователей стало много, просто увеличьте количество ИИ-воркеров:
code
Bash
docker-compose up -d --scale ai-worker=3
RabbitMQ сам распределит задачи между тремя "мозгами".
Как улучшить:
Смена модели: Просто поменяйте CURRENT_LLM_MODEL в .env на gpt-4o-mini (дешевле) или claude-3-5-sonnet (умнее). Перезапуск не требуется (воркер подхватит при следующем запросе).
Добавление инструментов:
Создайте файл в app/ai_worker/tools/.
Импортируйте и добавьте его в список tools_list в agent.py.
Опишите его в system_prompt.md.
RAG: Можно добавить инструмент upload_document, который будет принимать PDF, разбивать на чанки и сохранять в vector_knowledge.
7. Безопасность
Whitelist: Только пользователи из ALLOWED_USERS могут писать боту.
Proxy: Весь внешний трафик (OpenAI, Telegram) скрыт за Squid-прокси.
Database: Порт 5432 открыт наружу в docker-compose, для реального продакшена на открытом сервере лучше его закрыть (удалить секцию ports у сервиса db).
Проект готов к запуску. Start and Go! 🚀
